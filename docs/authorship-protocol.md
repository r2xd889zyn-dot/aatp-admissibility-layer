# AATP Authorship Protocol

## Purpose

Defines how authored content is treated within AATP.

---

## Principle

If a signal has an author, it carries intent and influence.

Therefore:

→ it must be evaluated before it is allowed to become context

---

## Rule

No authored content is admitted directly.

All authored signals must pass through:

1. Authorship Classification  
2. Admissibility Evaluation  
3. Anchoring  

---

## Authorship Tiers

Tier 1 — Verified Author  
→ known, trusted, or self-authored

Tier 2 — Attributed Author  
→ named but not verified

Tier 3 — Unknown / Anonymous  
→ no clear origin

---

## System Flow

External Knowledge
→ Authorship (Protocol)
→ Admissibility
→ Anchoring  
→ Context  
→ Reasoning  

---

## Doctrine

Authorship defines who has influence.

Admissibility defines what is allowed.

Together, they define the system’s boundary of reality.
